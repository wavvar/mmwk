# 模组参考文档设计说明

## 概述

本说明定义了如何在 `modules/` 目录下新增 6 份中文模组参考文档：

- `wdr-m_cn.md`
- `mini_cn.md`
- `pro_cn.md`
- `ml6432a_cn.md`
- `ml6432a_bo_cn.md`
- `wdr-4g_cn.md`

新文档将以 `modules/ml6432ax_cn.md` 的写作风格和结构习惯为基准，但每个文件都允许根据当前资料覆盖情况裁剪章节。凡是现有仓库资料无法明确支撑的内容，包括管脚定义，都直接留空，不从系列级通用描述中推断补写。

## 目标

- 让这 6 份新文档在结构和呈现风格上与 `modules/` 现有模组文档保持一致
- 只复用当前 `modules/` 目录下已有文件中可追溯的内容
- 仅在仓库里存在明确文字依据时填写目标型号的管脚定义
- 对缺失内容保持留空，方便后续人工补充

## 非目标

- 不为这批文件重新设计一套新的文档格式
- 不在来源不明确时，把系列级通用描述推断成某个具体型号的结论
- 不引入外部资料或联网检索内容
- 不把只有图片、没有文字依据的 pinout 图强行转换成新的文字定义

## 备选方案

### 1. 弹性主模板方案

以 `modules/ml6432ax_cn.md` 为主模板，统一写作风格和章节骨架，再按目标型号的资料情况裁剪不适用章节。

优点：

- 六份文档整体风格统一
- 可以避免资料不足的文档沦为空壳
- 符合用户“统一模板风格，但允许裁剪”的选择

缺点：

- 不同文件的章节数量不会完全一致

### 2. 严格模板复制方案

六份文档全部复制同一套章节结构，无法支撑的部分直接留空。

优点：

- 结构最整齐

缺点：

- 会生成多份信息密度很低的空章节文档
- 对 `wdr-m_cn.md`、`wdr-4g_cn.md`、`mini_cn.md`、`pro_cn.md` 尤其不合适

### 3. 按型号自由重组方案

每个文件完全按照该型号现有资料组织章节，不强求与模板对齐。

优点：

- 最贴近原始资料覆盖情况

缺点：

- 六份文档会缺乏统一性
- 后续维护和补充成本更高

## 选定方案

选择方案 1。

每个新文件都继承 `modules/ml6432ax_cn.md` 的语气、标题风格、表格样式、图片使用方式和相对链接习惯。无法支撑的章节可以删除；保留下来的章节中，无法支撑的内容直接留空。

## 内容来源优先级规则

每个目标文件的内容按以下优先级选取：

1. 明确点名该目标模组或板卡的文字内容
2. 同系列中能够明确直接适用于该目标型号的文字内容
3. 可以直接复用、且不需要新增解释文字的现有图片引用
4. 当仓库没有明确支持时，直接留空

这一规则同时适用于管脚定义和非管脚内容。

## 留空规则

- 缺失内容直接留空
- 不写 `TODO`、`TBD`、`待补充` 等占位提示
- 可以接受空段落、空表格单元格或最小结构的空表格
- 留空区域应保持便于后续人工直接编辑

## 文档约定

- 语言：中文
- 输出格式：Markdown
- 输出位置：`modules/`
- 文件名必须与用户指定名称完全一致
- 图片和文档链接继续沿用仓库现有相对路径习惯
- 章节命名在适用时尽量贴近 `modules/ml6432ax_cn.md`

## 各文件设计

### `ml6432a_cn.md`

主要来源：

- `modules/ml6432ax_cn.md`
- `modules/mdr_cn.md`

计划写入内容：

- `ML6432A` 的模组简介
- 明确属于该型号的尺寸和图片
- 适用于 `ML6432A` 的接口图片与参考说明
- 仅在来源明确共享时复用烧录与使用章节

计划留空或省略：

- 所有 BO 专属内容
- 当前资料里没有明确文字依据的该型号管脚定义

### `ml6432a_bo_cn.md`

主要来源：

- `modules/ml6432ax_cn.md`
- `modules/mdr_cn.md`

计划写入内容：

- `ML6432A_BO` 的模组简介
- BO 版本对应的尺寸和图片
- 来源中明确存在的直插集成说明
- BO 版本对应的接口线序图片和参考说明

计划留空或省略：

- 无法干净拆分出来的非 BO 内容
- 没有明确文字依据的管脚细节

### `wdr-m_cn.md`

主要来源：

- `modules/mdr_cn.md`
- `modules/README_CN.md`

计划写入内容：

- `WDR-M` 作为主控承载板的板卡简介；在系统文档中对应 `MDR-M`
- 其在 WDR 系统中的角色说明
- 与 `ML6432A`、`ML6432A_BO` 的兼容关系说明
- 现有的 USB Type-C、LED、按键、插接方向等板级参考图

计划留空或省略：

- 当前资料中未明确给出的板卡技术规格
- Markdown 文字中没有明确写出的管脚定义

### `wdr-4g_cn.md`

主要来源：

- `modules/mdr_cn.md`
- `modules/README_CN.md`

计划写入内容：

- `4G Cat1` 通信板在 WDR 系统中的板卡简介
- 远程联网与系统回传中的角色说明
- 现有连接关系图和接口原理参考图

计划留空或省略：

- 当前资料中不存在的详细板卡规格
- Markdown 文字中没有明确提供的管脚定义

### `mini_cn.md`

主要来源：

- `modules/rpx_cn.md`

计划写入内容：

- `MINI` 模组简介
- 当前已有的 `MINI` 图片
- 文档中明确提到 `Mini` 的开发调试方向说明
- 当前文档已经明确定义的共用烧录器 Type-C 引脚说明

计划留空或省略：

- 不能安全归属到 `MINI` 单型号的规格项
- 没有明确提供的模组管脚定义

### `pro_cn.md`

主要来源：

- `modules/rpx_cn.md`

计划写入内容：

- `PRO` 模组简介
- 当前已有的 `PRO` 图片
- 文档中明确提到 `Pro` 的开发调试方向说明
- 当前文档已经明确定义的共用烧录器 Type-C 引脚说明

计划留空或省略：

- 不能安全归属到 `PRO` 单型号的规格项
- 没有明确提供的模组管脚定义

## 验收方案

在实施完成前，按以下标准自检：

- 6 个目标文件都已创建在 `modules/` 下
- 文件名与用户要求完全一致
- 已填写内容都能追溯到现有 `modules/` 资料
- 没有编造新的管脚定义
- 不存在 `TODO`、`TBD`、`待补充` 之类占位词
- 相对链接和图片路径都指向仓库中真实存在的资源

## 风险

- `WDR-M`、`WDR-4G`、`MINI`、`PRO` 的独立资料覆盖明显弱于 `ML6432A` 和 `ML6432A_BO`
- 部分可用的管脚或连接器信息只存在于图片中，若不做人工判读，能落成的文字内容会有限
- 最终 6 份文件会保持风格统一，但章节数量和信息密度不会完全一致

## 实施边界

本说明只覆盖文档创建，不包含对仓库现有信息之外的硬件细节进行补全或扩展。
